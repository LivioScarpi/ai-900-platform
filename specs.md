# AI-900 Study Platform — Specs

## Overview

A web application to help study for the Microsoft AI-900 certification exam. The app is powered by a **pre-processed JSON file** (`questions.json`) generated by a one-time preprocessing script. The script reads the source Markdown file and uses Claude vision API to intelligently extract structured data from images. The resulting JSON is committed to the repo and used directly by the app — no image processing at runtime.

The app is built with **Next.js 14 + TypeScript**, styled with **Tailwind CSS**, deployed on **Vercel**, and uses **Supabase** (free tier) for persistent result tracking.

---

## Architecture: Two-Phase Approach

### Phase 1 — Preprocessing (runs once, locally, before deployment)

A Node.js script (`scripts/preprocess.ts`) that:
1. Reads `questions.md` and parses raw question objects
2. For each question containing images, fetches them from GitHub raw URLs and calls **Claude vision API** to classify and extract structured data from each image
3. Outputs a fully structured `data/questions.json`
4. This file is committed to the repo — it never needs to run again unless the question bank changes

### Phase 2 — The App (reads only from questions.json)

Next.js app that reads `questions.json` at build time. No image processing, no Claude API calls at runtime. Fast, static-friendly, deployable to Vercel.

## Image Source

Images are hosted on GitHub:
```
https://raw.githubusercontent.com/anxkhn/azure-ai-900-exam-prep/main/images/<filename>.jpg
```

The preprocessing script fetches them directly from these URLs. Images do not need to be downloaded or committed to this repo.

For `CONTEXT_IMAGE` types (see below), the GitHub raw URL is stored in the JSON and used directly by the app to display the image at runtime.

## Preprocessing Script (`scripts/preprocess.ts`)

### Step 1 — Parse the Markdown

Extract all questions from `questions.md` using regex. For each question collect:
- Question number
- Raw question text (may contain `![](images/filename.jpg)` references)
- Raw options text (A, B, C, D... — may be empty for image-based questions)
- Answer(s) string (e.g. `B` or `A,C,E`)
- Explanation text (may also contain image references)
- Reference URLs

Pure text MCQ questions (no images) are fully structured from the MD alone — no API call needed.

### Step 2 — Classify every image

For each image found in a question or its explanation, fetch it from the GitHub raw URL and send it to **claude-sonnet-4-20250514** (vision) with this classification prompt:

```
You are processing images from a Microsoft AI-900 exam question bank.
Classify this image into exactly ONE of these roles:

1. CONTEXT_IMAGE — A visual that is part of the question content and must be shown to the user
   (e.g. a confusion matrix, a diagram, a chart, a screenshot of a UI, a table of data).
   The user needs to see this image to understand and answer the question.

2. QUESTION_SHELL — The image contains the question/answer UI rendered as an image
   (e.g. a "Hot Area" box with dropdown selects, a Yes/No table, a drag-drop matching layout).
   Extract all text content so it can be rendered as interactive UI components.

3. EXPLANATION_IMAGE — The image shows the completed/correct answer or explanation
   (e.g. filled-in version of the Hot Area, annotated diagram with answers).
   Extract all text content so it can be shown as a plain text explanation.

Respond in JSON only.
```

### Step 3 — Extract structured content per image role

#### CONTEXT_IMAGE
Keep as an image. Store the GitHub raw URL. No text extraction needed.
```json
{ "role": "CONTEXT_IMAGE", "imageUrl": "https://raw.githubusercontent.com/..." }
```

#### QUESTION_SHELL
Extract the interactive structure. Claude must identify the subtype and return the appropriate shape:

**Subtype: SENTENCE_COMPLETION** — one dropdown completing a sentence
```json
{
  "role": "QUESTION_SHELL",
  "subtype": "sentence_completion",
  "sentence": "The process of [BLANK] involves training a model on labeled data.",
  "options": ["supervised learning", "unsupervised learning", "reinforcement learning"],
  "correctAnswer": "supervised learning"
}
```

**Subtype: MULTI_STATEMENT_YESNO** — table of statements, each Yes or No
```json
{
  "role": "QUESTION_SHELL",
  "subtype": "multi_statement_yesno",
  "statements": [
    { "text": "Anomaly detection can identify fraudulent transactions.", "correct": "Yes" },
    { "text": "Clustering is a supervised learning technique.", "correct": "No" }
  ]
}
```

**Subtype: MULTI_STATEMENT_DROPDOWN** — table of statements, each with a dropdown
```json
{
  "role": "QUESTION_SHELL",
  "subtype": "multi_statement_dropdown",
  "statements": [
    {
      "text": "A system that predicts whether an email is spam uses [BLANK].",
      "options": ["classification", "regression", "clustering"],
      "correctAnswer": "classification"
    }
  ]
}
```

**Subtype: DRAGDROP** — match items from left column to targets on right column
```json
{
  "role": "QUESTION_SHELL",
  "subtype": "dragdrop",
  "items": ["Computer Vision", "Natural Language Processing", "Anomaly Detection"],
  "targets": [
    { "text": "Detecting unusual patterns in network traffic", "correctItem": "Anomaly Detection" },
    { "text": "Extracting key phrases from customer reviews", "correctItem": "Natural Language Processing" },
    { "text": "Identifying objects in a photo", "correctItem": "Computer Vision" }
  ]
}
```

#### EXPLANATION_IMAGE
Extract visible text and annotated answers as a plain string.
```json
{
  "role": "EXPLANATION_IMAGE",
  "text": "Box 1: Yes — Anomaly detection can identify unusual patterns...\nBox 2: No — Clustering is unsupervised..."
}
```

### Step 4 — Assemble the final question object

After classifying all images, combine the extracted data with the parsed MD text to produce the final typed Question object. Write all questions to `data/questions.json`.

## Final Question Schema (`data/questions.json`)

```ts
type Question =
  | McqQuestion
  | MultiQuestion
  | SentenceCompletionQuestion
  | YesNoQuestion
  | DropdownQuestion
  | DragDropQuestion;

interface BaseQuestion {
  id: number;
  topic: Topic;
  explanation: string;      // plain text (from MD + extracted from EXPLANATION_IMAGEs)
  reference?: string;
  contextImages: string[];  // GitHub raw URLs of CONTEXT_IMAGE images — shown above question
}

interface McqQuestion extends BaseQuestion {
  type: "mcq";
  text: string;
  options: { letter: string; text: string }[];
  correctAnswer: string;    // e.g. "B"
}

interface MultiQuestion extends BaseQuestion {
  type: "multi";
  text: string;
  options: { letter: string; text: string }[];
  correctAnswers: string[]; // e.g. ["A", "C", "E"]
}

interface SentenceCompletionQuestion extends BaseQuestion {
  type: "sentence_completion";
  sentence: string;         // contains [BLANK] placeholder
  options: string[];
  correctAnswer: string;
}

interface YesNoQuestion extends BaseQuestion {
  type: "yesno";
  text: string;
  statements: {
    text: string;
    correct: "Yes" | "No";
  }[];
}

interface DropdownQuestion extends BaseQuestion {
  type: "dropdown";
  text: string;
  statements: {
    text: string;
    options: string[];
    correctAnswer: string;
  }[];
}

interface DragDropQuestion extends BaseQuestion {
  type: "dragdrop";
  text: string;
  items: string[];          // draggable chips
  targets: {
    text: string;
    correctItem: string;
  }[];
}
```

## Interactive UI Components (per question type)

### `<McqCard>` / `<MultiCard>`
Standard lettered option buttons. Single select highlights on click; multi-select allows toggling multiple. On confirm: correct = green ✅, selected wrong = red ❌, and correct option always highlighted green.

### `<SentenceCompletionCard>`
Renders the sentence with a styled `<select>` dropdown replacing `[BLANK]`. On confirm: the inline dropdown is highlighted green (correct) or red (wrong) with the correct answer shown.

### `<YesNoCard>`
Renders a table. Each row: statement text + Yes / No toggle buttons. On confirm: each row turns green or red.

### `<DropdownCard>`
Renders a table. Each row: statement text + a `<select>`. On confirm: each row turns green or red with correct answer shown.

### `<DragDropCard>`
Two-column layout using `@dnd-kit/core`:
- Left column: draggable chips (the `items` array)
- Right column: drop zones, one per target (showing the `target.text`)

User drags chips into drop zones. Can re-drag to reassign. A "Check Answers" button evaluates all mappings: correct = green, incorrect = red + shows the correct item label.

### `<ContextImage>`
Renders a GitHub raw image URL. Used in any card type when `contextImages` is non-empty. Shown above the question text in a rounded card. Click-to-zoom via a modal/lightbox.

### `<ExplanationDrawer>`
Collapsible panel shown after answering any question type. Contains explanation text + reference link. Consistent across all card types.

## Study Modes

### 1. Sequential Review
All 132 questions in original file order. No timer. Answer → reveal correct + explanation → Next/Prev. Progress bar: "34 / 132".

### 2. Random Shuffle
Same as Sequential but order randomized once at session start. Seed stored in `sessionStorage` — refresh doesn't reshuffle mid-session.

### 3. Topic Practice
User picks one or more topics from a list. Only matching questions shown in random order. Topic badge visible on every card.

### 4. Exam Simulator (50 questions)
- 50 questions drawn proportionally across all topics, no duplicates
- 45-minute countdown timer at the top
- No per-question feedback during exam
- On submit: score, per-topic breakdown, full review of wrong answers
- Results saved to Supabase

### 5. Flashcard Mode
- Show question text (+ context images if any), no options visible
- User thinks, then clicks "Reveal Answer" to see the correct answer + explanation
- Self-rate: ✅ Got it / ❌ Missed it
- End-of-deck summary
- Ratings saved to Supabase

### 6. Quick Fire
- 10 random questions, 15 seconds each
- Timer bar per question — auto-advances on timeout (counts as wrong)
- Immediate green/red feedback
- Final score at end

### 7. Weak Spots Review
- Queries Supabase for questions with wrong-answer rate > 50% (minimum 2 attempts)
- Sorted worst-first
- Graceful fallback: "Answer more questions first!" if insufficient data

### 8. Bookmarked Questions
- 🔖 icon on every question card
- Bookmarks saved/removed in Supabase
- Dedicated mode shows only bookmarked questions

## Supabase Schema

Anonymous user ID generated once in the browser, stored in `localStorage`. No auth required.

```sql
create table attempts (
  id uuid primary key default gen_random_uuid(),
  user_id text not null,
  question_id int not null,
  mode text not null,
  selected_answers text[],
  is_correct boolean not null,
  time_taken_ms int,
  created_at timestamptz default now()
);

create table exam_sessions (
  id uuid primary key default gen_random_uuid(),
  user_id text not null,
  mode text not null,
  score int not null,
  total int not null,
  topic_scores jsonb,   -- { "responsible_ai": { correct: 3, total: 5 }, ... }
  duration_ms int,
  created_at timestamptz default now()
);

create table flashcard_ratings (
  id uuid primary key default gen_random_uuid(),
  user_id text not null,
  question_id int not null,
  rating text not null,  -- 'got_it' | 'missed_it'
  created_at timestamptz default now()
);

create table bookmarks (
  id uuid primary key default gen_random_uuid(),
  user_id text not null,
  question_id int not null,
  created_at timestamptz default now(),
  unique(user_id, question_id)
);
```

## Running the Preprocessor

```bash
# Install dependencies
npm install

# Set your Anthropic API key
export ANTHROPIC_API_KEY=sk-ant-...

# Run once — outputs data/questions.json
npx ts-node scripts/preprocess.ts

# Commit the result — the app never needs to run this again
git add data/questions.json
git commit -m "chore: add preprocessed question bank"
```

The script logs progress per question and skips images-less questions (pure text MCQs need no API call). Estimated Claude API cost: minimal — only ~50-60 images require vision analysis across the entire question bank.
